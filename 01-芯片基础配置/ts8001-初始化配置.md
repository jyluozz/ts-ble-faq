# TS8001 初始化配置（FAQ）

## 问题现象
- 新板上电后不广播 / 不可连接 / 初始化失败

## 影响范围
- **芯片**：TS8001
- **SDK/固件**：填写版本号

## 典型根因
- 时钟/电源/IO 复用配置不匹配
- BLE 协议栈未正确初始化或初始化顺序错误
- 编译宏/链接脚本导致关键段缺失

## 排查步骤（从易到难）
1. **确认供电与复位**：电压、上电时序、复位脚、晶振是否起振（如适用）。
2. **确认基础日志**：是否能输出启动 banner / 版本信息 / assert 信息。
3. **确认 BLE 初始化流程**：初始化 API 调用顺序、返回值、是否有错误码。
4. **确认 RF/天线与匹配**：近距离也无法发现广播时，排查 RF 前端与天线匹配。

## 解决方案
- **推荐初始化顺序**：先完成系统时钟与外设基础初始化，再初始化 BLE 协议栈，最后启动广播/连接流程。
- **常见注意项**：
  - 低功耗相关配置建议在“能稳定广播/连接”后再逐步引入，避免把问题叠加。
  - 若使用外部 32k/晶振，需确认对应配置项一致。

## 验证方法
- 用手机/PC BLE 扫描工具能稳定发现广播（RSSI 合理、间隔稳定）
- 连接后可完成基础 GATT 读写（参考 `03-协议与数据交互/ts-特征值读写示例.md`）
