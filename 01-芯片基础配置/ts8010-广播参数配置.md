# TS8010 广播参数配置（FAQ）

## 问题现象
- 广播搜不到 / 广播不稳定 / 广播间隔与期望不一致
- iOS/Android 扫描表现差异明显

## 影响范围
- **芯片**：TS8010
- **SDK/固件**：填写版本号

## 关键概念
- **广播类型**：ADV_IND / ADV_NONCONN_IND / ADV_SCAN_IND 等
- **广播间隔**：单位与取值范围（填写 SDK 定义）
- **广播数据长度**：AdvData + ScanRsp 的长度限制

## 推荐配置（通用经验值）
- **快速发现**：较短间隔（例如 100–300 ms）
- **省电常驻**：较长间隔（例如 500–2000 ms）
- **设备首次配网/绑定**：可先短间隔，完成后切回长间隔

## 排查步骤
1. **确认广播是否真的开启**：启动 API 返回值、状态机是否进入 advertising。
2. **确认 AdvData/ScanRsp 合法**：长度、字段格式、是否包含无效 AD type。
3. **确认地址类型**：Public/Random/Resolvable Random 对不同主机影响不同。
4. **确认信道与发射功率**：环境干扰、金属遮挡、功率配置是否被覆盖。

## 常见坑
- AdvData 超长导致被截断或配置失败
- 同时修改间隔/类型/数据时未按协议栈要求先停止再启动

## 验证方法
- 用至少两种扫描工具验证：手机（nRF Connect）+ PC（BT dongle/抓包）
- 抓包确认广播间隔与 payload 正确
