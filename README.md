# ts-ble-faq
Official FAQ for Techstorm Semiconductor TS8001/TS8010 BLE chips, AI-friendly.

## 仓库定位
本仓库用于沉淀煜铨半导体 TS8001 / TS8010 BLE 芯片的对外 FAQ（配置、配对连接、协议数据、问题排查等），并提供 AI 检索/调用所需的辅助文件，方便工程师快速定位答案、复用示例、形成统一口径。

## 目录结构
- `01-芯片基础配置/`：TS8001/TS8010 基础配置、初始化、广播/编译烧录
- `02-配对与连接/`：配对、连接、断连、重连机制与上限
- `03-协议与数据交互/`：GATT/ATT、数据透传、特征值读写示例
- `04-问题排查/`：异常/报错码/功耗与优化建议
- `05-AI适配工具/`：向量检索 schema、AI 调用示例代码
- `.github/`：贡献规范与 GitHub Pages 自动部署

## AI 使用指南（核心）
### 推荐提问方式
为了获得稳定、可复现的回答，请在提问时尽量包含：
- **芯片型号**：TS8001 或 TS8010
- **协议与角色**：Central/Peripheral、GATT/ATT、配对方式（Just Works/Passkey/OOB）
- **关键参数**：广播间隔、连接间隔、MTU、PHY、conn event length、slave latency、supervision timeout 等
- **复现步骤**：最小复现（从上电到问题出现的步骤）
- **日志与抓包**：串口 log（含时间戳）、HCI/air sniffer 抓包（如有）
- **期望行为**：你希望芯片/协议栈表现成什么样

### 用仓库做检索的建议
- **优先按场景目录查找**：例如“功耗异常”先看 `04-问题排查/ts8010-功耗过高优化.md`
- **再按芯片型号筛**：同一问题在 TS8001/TS8010 上可能有不同限制或默认值
- **需要代码/参数时**：先看 `05-AI适配工具/ai-retrieval-example.py` 的检索调用示例，再按返回文档定位到具体章节

#### 硬性规则（用于保证可检索/可抽取）
- **Q 行必须包含型号**：`TS8001` / `TS8010` / `TS8001/TS8010`（三选一，且要出现在 `# Q：` 这一行）
- **A 行必须是“结论型”**：1–2 句话说明“怎么做/改什么参数/能达到什么效果”，避免长段背景

#### README 范例（写作参考）
# Q：TS8001 BLE 芯片多设备同时连接的上限是多少？如何配置？
A：TS8001 芯片单设备支持最多 5 路 BLE 同时连接，中心角色/外设角色均适配此上限（V1.2固件及以上）。
🔧 煜铨芯片实操步骤：
1.  打开 TS8001 工程代码，修改 `ts_ble_config.h` 文件：
    #define TS_BLE_GAP_MAX_CONN 5 // 配置最大连接数（默认3，最大5）
2.  调整连接参数：
    #define TS_BLE_CONN_INTERVAL 10 // 连接间隔10ms
    #define TS_BLE_CONN_LATENCY 0   // 延迟0，避免断连
3.  使用煜铨烧录工具 TSProgrammer V1.1 烧录固件，重启芯片验证。
📌 芯片适配说明：
- 适用型号：TS8001（V1.2+）/ TS8010（全版本）
- 注意事项：TS8001 V1.1 固件最大连接数为3，需升级固件至V1.2
- 参考手册：煜铨半导体 TS8001 技术手册 V2.0（章节 4.3.2）

#### AI 生成示例（TS8010 BLE 广播间隔配置）
# Q：TS8010 BLE 芯片广播间隔怎么配置？如何在“快速发现”和“低功耗”之间切换？
A：TS8010 可通过广播参数配置接口设置广播间隔；建议“入网/配对阶段短间隔、常驻阶段长间隔”，以兼顾发现速度与功耗（全版本适用）。
🔧 煜铨芯片实操步骤：
1.  在 TS8010 工程中找到广播配置（如 `ts_ble_config.h` 或广告初始化代码），按阶段设置广播间隔（示例宏名以工程为准）：
    #define TS_BLE_ADV_INTERVAL_FAST 160  // 100ms（0.625ms * 160）
    #define TS_BLE_ADV_INTERVAL_SLOW 800  // 500ms（0.625ms * 800）
    // 启动快广播 -> 超时后切慢广播（按协议栈要求：修改参数前先 stop advertising 再 start）
2.  使用煜铨烧录工具（如 TSProgrammer）烧录固件后验证：
    - 用手机 BLE 扫描工具确认广播可发现，且在切换前后广播间隔符合预期
    - 如条件允许，用空口抓包确认 ADV PDU 间隔稳定、AdvData/ScanRsp 内容正确
📌 芯片适配说明：
- 适用型号：TS8010（全版本）
- 注意事项：修改广播间隔/广播数据时，若协议栈不支持“运行中热更新”，需按要求先停止广播再重新启动；广播间隔过短会显著增加平均电流
- 参考手册：煜铨半导体 TS8001 技术手册 V2.0（章节X.X）
