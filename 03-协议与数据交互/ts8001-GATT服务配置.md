# TS8001 GATT 服务配置（FAQ）

## 问题现象
- 手机发现服务不全 / 特征值读写失败 / Notify 不生效
- iOS 能用但 Android 不行（或相反）

## 影响范围
- **芯片**：TS8001
- **角色**：Peripheral（典型：设备提供 GATT 服务）

## 基础要点
- **服务/特征值/描述符**：UUID、属性（read/write/notify/indicate）、权限（加密/认证）
- **CCCD**：Notify/Indicate 必须有 CCCD 且由客户端写入启用
- **MTU**：影响单包负载与分包策略

## 排查步骤
1. **确认服务是否注册成功**：注册 API 返回值、handle 范围是否合理。
2. **确认属性与权限**：客户端是否需要先配对/加密；权限不满足会导致 ATT error。
3. **确认 CCCD 流程**：客户端是否写入了 CCCD；服务端是否在回调中记录启用状态。
4. **确认分包**：长写/长读/notify 超过 MTU 时是否做了分包与重组。

## 解决方案建议
- **先做最小 GATT**：1 个服务 + 1 个可读写特征值 + 1 个 notify 特征值，跑通后再扩展。
- **权限逐步增强**：先明文可读写，验证通路后再加密/认证要求。

## 验证方法
- 使用 BLE 工具完成：发现服务 -> 读特征值 -> 写特征值 -> 打开 notify -> 收到通知
- 抓包确认 ATT opcode/handle/错误码符合预期
