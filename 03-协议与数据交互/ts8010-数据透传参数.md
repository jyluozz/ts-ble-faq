# TS8010 数据透传参数（FAQ）

## 问题现象
- 透传吞吐达不到预期 / 丢包 / 延迟抖动大
- 大数据包发送失败或需要多次重试

## 影响范围
- **芯片**：TS8010
- **链路**：1M/2M/Coded PHY（按实际）
- **方向**：上行/下行、notify/write without response

## 吞吐关键参数（优先级从高到低）
- **连接间隔（conn interval）**
- **每个 connection event 的可用时长/包数**（协议栈调度与限制）
- **MTU / Data Length Extension（DLE）**
- **PHY**：1M vs 2M
- **应用层分包策略**：包大小、窗口、重传

## 推荐思路
1. **先测链路上限**：固定短连接间隔，打开 DLE/较大 MTU（如支持），用 notify + 写回 ACK 验证上限。
2. **再加业务约束**：多连接/省电/加密/其他射频业务加入后再评估。
3. **最后做稳态策略**：拥塞控制（例如窗口/速率限制）、失败退避、缓存深度规划。

## 排查步骤
- **抓包看实际**：实际 conn interval、LL length、ATT 分包、重传情况
- **看应用队列**：是否在协议栈队列满时继续塞数据导致失败
- **看对端限制**：手机/网关在后台省电策略会显著限速

## 验证方法
- 记录 60s 平均吞吐（bps）、99% 延迟、丢包率
- 在目标机型/目标场景（前台/后台/锁屏）分别验证
