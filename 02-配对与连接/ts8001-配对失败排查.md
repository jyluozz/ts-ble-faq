# TS8001 配对失败排查（FAQ）

## 问题现象
- 手机提示“配对失败/无法配对”
- 连接成功但启用加密失败（bonding 不生效）
- 反复弹窗请求配对，或配对后立刻断连

## 影响范围
- **芯片**：TS8001
- **主机**：iOS / Android / Windows（填写版本）
- **安全参数**：IO 能力、MITM、LESC、Key size、bonding

## 常见根因（按概率）
1. **安全参数不兼容**：两端 IO 能力/认证需求不匹配（比如一端强制 MITM 但无输入能力）。
2. **密钥存储/回调异常**：配对完成但未保存 LTK/IRK/CSRK，导致下次无法加密或重复配对。
3. **连接参数/超时过紧**：配对期间连接间隔太大或 supervision timeout 过小引发超时断连。
4. **GATT 流程干扰**：在配对完成前大量读写/通知，导致协议栈忙或触发安全失败。

## 排查步骤
1. **确认失败阶段**：是 Pairing Request 阶段，还是 Encryption Start 阶段失败（看日志/抓包）。
2. **对齐安全配置**：确认是否开启 LESC、是否要求 MITM、IO 能力选择是否合理。
3. **检查 bonding 数据**：是否有保存/恢复 bonding；是否区分不同 peer（按地址/IRK）。
4. **用抓包定位 SMP 错误码**：记录 Pairing Failed 的 reason。

## 解决方案（通用）
- **优先策略**：先做 Just Works + bonding 跑通，再逐步增加 MITM/Passkey/LESC。
- **配对期间降载**：配对完成前减少 GATT 操作与通知频率。
- **保证密钥持久化**：断电重启后仍能加密连接，避免重复配对。

## 验证方法
- 首次配对成功后，断开重连可直接加密（不再弹配对）
- 通过抓包确认 SMP 流程完整且无 Pairing Failed
